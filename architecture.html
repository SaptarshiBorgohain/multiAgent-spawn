TripWeave / Low-Level System Design

Complete Architecture Reference

Autonomous multi-agent travel orchestration. FastAPI backend, Next.js 14 frontend, Redis Streams task bus, PostgreSQL + Elasticsearch persistence, RestrictedPython custom-agent sandbox, magic-link auth, and Splitwise-style collaboration — all running in Docker Compose.

FastAPI (Python 3.12) Next.js 14 / React 18 Redis Streams + Pub/Sub PostgreSQL 15 + SQLAlchemy async Elasticsearch 8 RestrictedPython ≥ 7 DeepSeek LLM API Magic Link + JWT (HS256)

End-to-End Request Flow

1. Planner / Orchestrator app/backend/orchestrator/planner.py

Entry point for every trip. Parses intent via LLM, builds a dependency-ordered task DAG, publishes all tasks to the Redis Stream, then fires a planning_started Pub/Sub event with full metadata.

Fixed DAG topology

→
transport_planning
+
hotel_planning
+
food_discovery
↓
itinerary_optimization
→
budget_optimizer
→
custom:{uuid} ×N

Custom agents depend on budget_optimizer (last built-in stage). Multiple custom agents can run in parallel after that gate.

Task envelope (published to Redis Stream) 
{
  "task_id":        "uuid4",
  "session_id":     "<trip_id>",
  "task_type":     "hotel_planning",  // or "custom:{uuid}"
  "depends_on":    ["destination_research"],
  "context_scope": "<trip_id>",
  "intent": {
    "destination":   "Kyoto",
    "budget_inr":    "80000",
    "travel_style":  "cultural",
    "duration_days": "7",
    "model":         "deepseek-v4-flash"
  }
}
_extract_intent() Calls DeepSeek (deepseek-v4-flash, response_format: json_object) to extract destination / budget_inr / travel_style / duration_days from free-form query. _build_dag() Sorts task types by TASK_DEPENDENCY_MAP, appends custom:{ca_id} with dep_map["custom:{id}"] = ["budget_optimizer"]. Returns list of task dicts. handle_budget_exceeded() Autonomous replanning: re-publishes a hotel_planning task with intent.budget_constraint = "cheaper" and fires {"type":"replanning","reason":"budget_exceeded"} event. monitor_and_replan() Reads agent statuses from Redis, calls handle_budget_exceeded if cost threshold breached.

3. Dynamic Worker Loop app/backend/workers/worker.py

Ephemeral by design — worker_runner.py spawns a new coroutine per worker. Each worker blocks on the Redis Stream, checks deps, dispatches via TOOL_REGISTRY, then ACKs.

xreadgroup block=5s
→
JSON.loads(payload)
→
_check_dependencies()
→
ready / wait / cancel
TOOL_REGISTRY[task_type]
→
set_agent_status("running")
→
await handler(task)
→
set_agent_status("done") + publish_event + xack
wait path Re-publishes the task (publish_task(task)) then ACKs the original delivery to prevent PEL pile-up. asyncio.sleep(1) backoff. cancel path Marks task failed, publishes task_failed event with failed dep names. Cascade-fails all downstream tasks. custom task dispatch if task_type.startswith("custom:"): handler = run_custom_agent — dynamic lookup, not in static TOOL_REGISTRY. TOOL_REGISTRY destination_research · transport_planning · hotel_planning · food_discovery · itinerary_optimization · budget_optimizer · places_search

5. Tool Handlers — LLM Calls app/backend/workers/tools.py

destination_research

Parallel execution: asyncio.gather(places_search_coro, llm_coro)

  • Places: attractions_index
  • LLM: deepseek-v4-flash, JSON response {highlights, best_areas, tips}
  • Writes: attractions + destination_highlights to context

itinerary_optimization

Reads full context; calls deepseek-chat (reliable JSON). Returns day-by-day plan.

  • Model: deepseek-chat (not flash — structured output)
  • Prompt includes actual venue names from context
  • Day schema: day, theme, morning, afternoon, evening, tips
  • Strips markdown fences if present

budget_optimizer

Reads finished itinerary + context. Calls deepseek-chat with temperature=0.3, max_tokens=600.

  • Returns: estimated_total_inr, breakdown{}, savings_tips[], verdict
  • Graceful: strips ```json fences; falls back to {"verdict": raw[:300]} on parse fail

6. Custom Agent Sandbox workers/tools.py · api/custom_agents.py

User-authored Python code runs inside a RestrictedPython jail. All subscript / iteration / write operations require explicit hooks. After the sandbox executes, the result is passed to DeepSeek as context for an LLM-enriched response.

Sandbox globals injected 
from RestrictedPython.Guards import guarded_iter_unpack_sequence
# NOTE: renamed from safe_iter_unpack_sequence in RestrictedPython >= 7.0

glb = {
  **safe_globals,
  "__builtins__": safe_builtins,
  "_getiter_":        iter,
  "_getitem_":        lambda obj, key: obj[key],  # obj["key"] access
  "_getattr_":        getattr,
  "_write_":          lambda obj: obj,            # assignment target
  "_inplacevar_":     lambda op, x, y: x+y if op=="+=" else x-y,
  "_unpack_sequence_": guarded_iter_unpack_sequence,
  "json":    json,
  "context": context,     # trip context dict (read-only by convention)
  "secrets": secrets,     # api_keys from CustomAgent.api_keys JSONB
  "http_get":  _safe_http_get,
  "http_post": _safe_http_post,
  "result":  {},            # user must set result = {...}
}

8. Expenses & Settlement api/expenses.py · utils/settlement.py

Expense lifecycle
  • POST /api/trips/{id}/expenses → creates Expense + ExpenseSplit rows
  • Splits are per-user amounts stored in expense_splits table
  • GET /api/trips/{id}/expenses/balances → aggregates net balance per user
  • GET /api/trips/{id}/expenses/settlement → calls minimize_cash_flow(net_dict)
  • POST /api/trips/{id}/expenses/{eid}/settle → marks Expense.settled = True
minimize_cash_flow() — greedy O(n²) 
# net = {"A": -300, "B": +200, "C": +100}
while b:
    creditor = max(b, key=lambda k: b[k])   # owed most
    debtor   = min(b, key=lambda k: b[k])   # owes most
    amount = min(b[creditor], -b[debtor])
    transactions.append({
        "from": debtor, "to": creditor, "amount": amount
    })
    # pruning: at most (n-1) transactions for n people
# → [{"from":"A","to":"B","amount":200},{"from":"A","to":"C","amount":100}]

9. Members & Invites app/backend/api/members.py

Membership model
  • TripMember(trip_id, user_id, role) — roles: owner | editor | viewer
  • POST /api/trips/{id}/members — owner adds user directly by user_id
  • DELETE /api/trips/{id}/members/{uid} — owner removes member
  • GET /api/trips/{id}/members — list with user details (join)
Invite token flow
  • TripInvite(trip_id, token, role, max_uses, expires_at)
  • POST /api/trips/{id}/invites — generates shareable token link
  • POST /api/invites/join — validates token expiry + max_uses, creates TripMember
  • Token is URL-safe random string; expiry and max-uses enforced at join time
